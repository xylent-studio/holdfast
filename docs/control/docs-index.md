# Holdfast Control Docs Index

These docs are not optional background reading. They are the working product contract for Holdfast.

## Read in this order
1. `product-charter.md`
2. `v1-scope.md`
3. `sync-offline-spec.md`
4. `interaction-model.md`
5. `state-and-transitions.md`
6. `ui-language-guide.md`
7. `agent-review-rubric.md`

## What each doc does
- **product-charter.md**: defines what Holdfast is, what it is not, and the product qualities that matter most
- **v1-scope.md**: defines what the first real version must and must not be
- **sync-offline-spec.md**: defines the sync-first experience and offline resilience rules
- **interaction-model.md**: defines how the app should behave screen-by-screen and action-by-action
- **state-and-transitions.md**: defines the product truth for item states and movements
- **ui-language-guide.md**: prevents dev-authored / AI-authored UI language
- **agent-review-rubric.md**: defines how to review changes and catch competent-but-wrong output

## Operating rule
If implementation, naming, or architecture conflicts with these docs, the conflict should be surfaced explicitly and resolved intentionally. Do not silently drift.
